Why This Project Exists

Blocking at one layer misses traffic. MetaBlock enforces policy at multiple control points so GitHub reviewers can inspect exactly how rules are generated and where they are applied.

AI-Assisted Rule Generation

A single generator builds browser rules, DNS configs, and metadata so your policy stays consistent.

Deterministic Outputs

Seed files are human-editable while generated files are machine reproducible for review and CI.

Operational Safety

If live ASN APIs fail, a maintained snapshot keeps blocking active instead of silently dropping coverage.

System Stack

1. Browser Layer

Chromium and Firefox manifests load generated declarativeNetRequest rules to stop domain requests.

2. DNS Layer

The local resolver blocks Meta domains and rejects DNS answers that map into blocked Meta prefixes.

3. Bot Layer

Robot signature output supports server-side blocks for known Meta crawler user-agents.

Launch In Minutes

Run this in the repository root:

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/generate_blocklists.py --skip-asn-fetch
python dns_service/server.py --port 5353
